# Cover Letter Guide

This guide is read by the agent during Phase 2 (Prepare). Only generates if `config.prepare.cover_letter_required` is true.

## Structure

3-4 paragraphs, concise. Never exceed one page.

### Paragraph 1: Hook
- State the role and company name
- 1-2 sentences connecting your strongest qualification to the role's core requirement

### Paragraph 2: Most Relevant Experience
- Pick the experience that best matches the JD
- Include specific accomplishments with metrics
- Mirror 2-3 JD keywords naturally

### Paragraph 3: Additional Value
- Second relevant experience, project, or skill set
- Show breadth — cover a different JD requirement than paragraph 2
- If company has a notable mission/product, reference it specifically

### Paragraph 4: Close
- Express enthusiasm for the specific company (not generic)
- One sentence on what you'd bring to the team
- "Best regards, [Name]"

## Rules

- **Never generic** — every letter must reference the specific company and role
- **Never fabricate** — only use experience from user-profile.md
- **Mirror JD language** — use the same terms the JD uses
- **Keep it short** — hiring managers skim. 250-400 words max
- **Match tone** — startup JD = conversational, enterprise JD = professional
- **No filler** — remove "I am writing to express my interest" type openings if space is tight
